# 🚀 Referencia Rápida - Comandos

## Comandos Disponibles

| Comando | Descripción | Uso |
|---------|-------------|-----|
| `/orquestador` | Enrutador inteligente (AUTO) | `/orquestador crear componente` |
| `/dev` | Angular Developer | `/dev crear servicio de API` |
| `/style` | Styling Agent | `/style implementa dark mode` |
| `/audit` | Auditor | `/audit revisa el código` |
| `/test` | Testing Agent | `/test crea tests unitarios` |
| `/help` | Ayuda de comandos | `/help` |

## Flujos Típicos

### 📱 Crear nueva feature
```
/orquestador crear un componente para [descripción]
```

### 🎨 Mejorar diseño
```
/style [tu solicitud]
```

### ✅ Validar código
```
/audit [tu solicitud]
```

### 🧪 Agregar tests
```
/test crear tests para [componente/servicio]
```

## Keywords de Auto-Routing

**Angular Developer:** component, service, route, module, angular, typescript
**Styling:** css, tailwind, estilo, diseño, responsive, dark mode
**Auditor:** revisar, bug, error, calidad, review, seguridad
**Testing:** test, prueba, spec, vitest, coverage

## Ejemplo Completo

```
# 1. Solicitud al orquestador
/orquestador crear componente de galería de proyectos con animaciones

# 2. El sistema detecta "componente" → Angular Developer
# El agente crea el componente

# 3. Luego solicitas estilos
/style añade efecto hover a las tarjetas de proyectos

# 4. Y finalmente tests
/test crea tests para el componente de galería
```

---

**Tip:** Empieza con `/orquestador` para que el sistema elija automáticamente. Si necesitas un agente específico, úsalo directamente.
