## 🤖 Comandos Disponibles en tu Orquestador

### 1️⃣ **Orquestador Principal**
```
/orquestador [tu solicitud]
```
**Uso:** Enruta automáticamente tu solicitud al agente correcto basándose en keywords.

**Ejemplo:**
```
/orquestador crear un nuevo componente para la sección de proyectos
```
El orquestador detectará "componente" y enrutará a **Angular Developer**.

---

### 2️⃣ **Acceso Directo a Agentes**

#### Dev (Angular Developer)
```
/dev [tu solicitud]
```
**Usa cuando:** Necesites crear componentes, servicios, rutas o cualquier feature Angular.

**Ejemplo:**
```
/dev crear un servicio de autenticación
```

#### Style (Styling)
```
/style [tu solicitud]
```
**Usa cuando:** Necesites diseño, CSS, Tailwind, dark mode o responsive.

**Ejemplo:**
```
/style implementa dark mode en la navbar
```

#### Audit (Auditor)
```
/audit [tu solicitud]
```
**Usa cuando:** Necesites revisar código, detectar bugs o auditar seguridad.

**Ejemplo:**
```
/audit revisa el componente de formulario de contacto
```

#### Test (Testing)
```
/test [tu solicitud]
```
**Usa cuando:** Necesites crear tests, coverage o validar testing.

**Ejemplo:**
```
/test crea tests para el servicio de autenticación
```

---

## 📊 Flujo Recomendado

1. **Planificar** → `/orquestador mi nueva feature es...`
2. **Desarrollar** → `/dev crear componente...` o `/dev crear servicio...`
3. **Estilizar** → `/style aplicar estilos...`
4. **Testear** → `/test crear tests para...`
5. **Revisar** → `/audit revisar código...`
